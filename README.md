# BDT Signal-Background Classifier for H → ZZ → 4μ

End-to-end binary classifier (signal vs. background) for the H → ZZ → 4μ search at √s = 13.6 TeV, trained in Python using xgboost on Monte Carlo samples produced with MadGraph5 + Pythia8 + Delphes.

```
NanoAOD-style ROOT  ─►  skim  ─►  features  ─►  train (xgboost)  ─►  evaluate
   (from MC gen)       uproot     helicity       BDT model           ROC, KS,
                       awkward    angles, pT      (.json)            importance
```

Input samples are generated by the companion repository:
[**madgraph**](https://github.com/siunkim0/madgraph) — MadGraph5 + Pythia8 + Delphes pipeline with NanoAOD conversion.

---

## Overview

| Parameter | Value |
|-----------|-------|
| Signal | ggH → ZZ → 4μ (HEFT, LO) |
| Background | qqZZ → 4μ, DY M > 50, tt̄ → 2ℓ2ν |
| √s | 13.6 TeV |
| Luminosity | 26.7 fb⁻¹ (2022 postEE equivalent weighting) |
| Framework | uproot → awkward → xgboost |
| Features | 17 kinematic + angular variables (5 HZZ helicity angles) |

The classifier is trained exclusively on MC. No collision data is used. Mass variables (`m4l`, `mZ1`, `mZ2`) are excluded from the training inputs to keep the BDT score decorrelated from the Higgs mass peak.

## Configuration

Two YAML files control the pipeline. **Do not hardcode paths or parameters in scripts** — edit these instead.

### `config/samples.yaml`

| Key | Description |
|-----|-------------|
| `lumi_fb` | Integrated luminosity (fb⁻¹) |
| `signal` / `background` | Per-sample: `path` (uproot glob to NanoAOD ROOT files), `xsec` (pb), `label` (1 = signal, 0 = background) |
| `data` | Data PD globs (unused in training) |

Fill in `path:` fields with the location of the NanoAOD-style ROOT files produced by the [madgraph](https://github.com/siunkim0/madgraph) pipeline.

### `config/selection.yaml`

| Key | Description |
|-----|-------------|
| `triggers` | HLT OR list |
| `muon`, `z_pairing`, `four_lepton` | Kinematic selection cuts |
| `bdt` | xgboost hyperparameters (n_estimators, max_depth, learning_rate, …) |
| `train` | Split fractions, RNG seed, GPU toggle |

## Pipeline

Each phase is a module under `src/`. Outputs of one phase are inputs to the next; each phase can be re-run independently as long as its inputs exist.

### Phase 1 — Skim (`src/skim.py`)

NanoAOD → flat per-event parquet. Applies HLT, muon ID/iso, 4-muon selection, Z₁/Z₂ pairing, m₄ℓ window.

```bash
python -m src.skim \
    --config config/samples.yaml \
    --selection config/selection.yaml \
    --out data/ntuples/ \
    --only ggH_ZZ_4l qqZZ_4l DY_M50 TTto2L2Nu
```

Output: `data/ntuples/<sample>.parquet`.

### Phase 2 — Features (`src/features.py`)

Appends engineered columns (pT ratios, angular separations, 5 HZZ helicity angles defined in [arXiv:1208.4018](https://arxiv.org/abs/1208.4018)) to the Phase 1 parquet files.

```bash
python -m src.features --in data/ntuples/ --out data/ntuples/
```

Output: `data/ntuples/<sample>_features.parquet`.

### Phase 3 — Train (`src/train.py`)

Loads all `*_features.parquet` files, builds class-balanced training weights (per-sample equal weighting rather than pure cross-section weighting), splits 65/10/25 train/val/test, fits xgboost with early stopping.

```bash
python -m src.train \
    --samples config/samples.yaml \
    --config  config/selection.yaml \
    --ntuples data/ntuples/ \
    --out     data/models/bdt_v1.json \
    2>&1 | tee logs/train_v1.log
```

Outputs:
- `data/models/bdt_v1.json` — xgboost native model.
- `plots/training_history.png` — train/val AUC vs. boosting round.

### Phase 4 — Evaluate (`src/evaluate.py`)

Reloads the dataset, recreates the same train/val/test split using the seed from `selection.yaml`, loads the trained booster, and produces diagnostic plots.

```bash
python -m src.evaluate \
    --model   data/models/bdt_v1.json \
    --ntuples data/ntuples/ \
    --samples config/samples.yaml \
    --config  config/selection.yaml \
    2>&1 | tee logs/evaluate_v1.log
```

Outputs:
- `plots/roc.png` — weighted + unweighted ROC.
- `plots/feature_importance.png` — gain-based.
- `plots/overtraining.png` — train vs. test score histograms + KS p-values.
- `reports/phase4_summary.md` — evaluation summary.

## Features

The training input list is defined in `src/features.py::FEATURES` (17 variables by default):

| Category | Variables |
|----------|-----------|
| Event kinematics | `pt4l`, `eta4l` |
| Z boson kinematics | `pt_Z1`, `pt_Z2` |
| Per-muon (pT-sorted) | `pt_mu1`–`pt_mu4`, `eta_mu1`–`eta_mu4` |
| Helicity angles | `cos_theta_star`, `cos_theta1`, `cos_theta2`, `Phi`, `Phi1` |

Mass variables (`m4l`, `mZ1`, `mZ2`) are stored in the parquet files for diagnostic use but excluded from training. This prevents the BDT from trivially learning the Higgs mass peak, which would sculpt the background m₄ℓ distribution in a downstream mass-window fit.

## ONNX Export

For deployment in a C++ framework, the trained xgboost model is exported to ONNX:

```bash
python -m scripts.export_onnx
python -m scripts.export_onnx --onnx-out /path/to/deployment/bdt_v1.onnx
```

The export script (`scripts/export_onnx.py`) converts the model, strips the ZipMap node for plain `(N, 2)` float output, and runs a numerical parity check (max |Δ probability| < 10⁻⁵) on the full dataset before writing. A 100-event validation CSV is also produced for C++/Python parity verification.

## Known Limitations

- **MC only** — no collision data is used. HLT, muon ID, and impact parameter selections all pass trivially on Delphes-generated samples.
- **No PU reweighting** — pileup effects are not modeled.
- **No lepton scale factors** — muon ID/iso/trigger efficiencies are not corrected.
- **Reducible backgrounds** — DY and tt̄ produce four-muon final states only through jet → μ fakes, which Delphes models poorly (see [madgraph README](https://github.com/siunkim0/madgraph#background-samples)).
- **LO cross sections** — all samples are generated at leading order.
